# Agentic Swarm (Zoe) for CrownSpin

This repository ships with a local agent orchestration layer inspired by the ``OpenClaw + Codex/Claude`` article you shared. The orchestrator—`Zoe`—lives inside `.clawdbot`. She keeps track of every task, enriches prompts with real context, spawns tmux-backed agents, waits for CI/reviews, and notifies you on Telegram when a PR is safe to merge.

## Key Concepts

| Layer | Role |
| --- | --- |
| Zoe (`start-agent.sh`) | Builds the context-aware prompt, creates a worktree, starts a tmux session with the chosen agent, writes the task into `.clawdbot/active-tasks.json`, and issues the completion event. |
| `.clawdbot/agent-runner.sh` | Runs the actual agent command inside the tmux session, streams everything to a log, updates the task registry upon exit, and triggers an OpenClaw event if notifications are enabled. |
| `.clawdbot/check-agents.sh` + `check-agents.py` (Ralph Loop V2) | Cron-driven monitor that reads the registry, checks tmux health, inspects GH PR state, marks tasks as `ready_for_review`, and hints when human attention is needed. |
| `.clawdbot/cleanup-orphans.sh` | Daily sweep that removes completed worktrees and annotates tasks as cleaned up.

## Context Zoe Uses

1. `README.md` – explains CrownSpin's features, structure, and build steps.
2. `AppStoreMetadata.md` – spells out marketing copy, store requirements, keywords, and review notes.
3. Git status – Zoe captures `git status -sb` at spawn time so the agent knows the current tree state.

Every prompt generated by Zoe combines those sources, your task summary, extra instructions, and an explicit `Definition of done` that mirrors your workflow (PR creation, CI, AI reviews, screenshots when UI changes).

## Starting an Agent

Run Zoe from the repository root with a single command:

```bash
git fetch origin
./.clawdbot/start-agent.sh \
  --task fix-haptic-balance \
  --description "Resolve the heavy pattern timing drift on watchOS 10" \
  --agent codex \
  --extra "Limit file changes to `HapticManager.swift`, `HapticPattern.swift`, and `CrownSpinApp.swift`."
```

- `--task` is the identifier that becomes the branch/worktree name.
- `--description` is woven into the prompt, so Zoe can remind Codex/Claude what the customer cares about.
- `--agent` selects Codex or Claude (default: Codex). Geminis are omitted per your request.
- `--extra` lets you inject guida lines (`Focus on these files`, `This is the customer request from AppStoreMetadata`, etc.).

Zoe automatically:

1. Stores the prompt under `.clawdbot/prompts/<task>.md` (so you can review what she told the agent).
2. Logs the tmux output to `.clawdbot/logs/<task>.log`.
3. Adds an entry to `.clawdbot/active-tasks.json` with the `definitionOfDone` checklist, context references, and notify flag.
4. Spawns a tmux session (`zoe-codex-fix-haptic-balance`) so you can attach (`tmux attach -t ...`) at any time.

## Monitoring (Ralph Loop V2)

`.clawdbot/check-agents.sh` is the improved Ralph Loop. Run it manually or via cron (`*/10 * * * * cd ~/work/CrownSpin && ./.clawdbot/check-agents.sh`). It:

- Reads every task from `.clawdbot/active-tasks.json`.
- Checks whether the tmux session is still alive.
- Queries `gh pr list --head <branch>` and `gh pr view` for the latest PR state.
- Marks the task as `ready_for_review` when a PR exists and CI reports success.
- Appends notes when a tmux session died unexpectedly so Zoe can retry with more context next time.
- Prints a concise summary line per task so you know which ones need attention.

Use this script as your heartbeat during busy days—Zoe does most of the work, and the summaries tell you when to open Telegram. If you want more proactive work, raise tasks based on Sentry/meeting notes and call `start-agent.sh` for each.

## Cleanup & Cron Hooks

`.clawdbot/cleanup-orphans.sh` removes worktrees and annotates tasks after they finish. Add a nightly cron or `launchd` job:

```
0 2 * * * cd /Users/xcv58/work/CrownSpin && ./.clawdbot/cleanup-orphans.sh >> .clawdbot/cleanup.log 2>&1
```

Similarly, monitor agents with the 10-minute cron above. Both scripts log to `.clawdbot/logs` and do not commit their outputs (see `.gitignore`).

## Notifications & Definition of Done

- When an agent finishes, `agent-runner.sh` updates its registry entry, sets `status` to `done`/`failed`, and triggers `openclaw system event --text "Zoe: <task> is done" --mode now`. That automatically wakes you on Telegram.
- Each task carries the same checklist: PR created, branch synced to `master`, CI passes (lint/types/unit/E2E), Codex review, Claude review, and screenshots when UI changes. Zoe and the monitor scripts use those expectations as success signals, even though you still make the final merge decision.
- There is no Gemini reviewer in this setup because you asked to opt out.

## Where to take this next

- Drop your own notes or meeting transcripts into `.clawdbot/notes/` and have Zoe include them by extending `build_context()` in `start-agent.sh`.
- Add more `--extra` context (customer quote, support ticket) when you spawn a task.
- Extend `.clawdbot/check-agents.py` to inspect GitHub comments or PR labels from AI reviewers.
- If you launch multiple agents simultaneously, monitor `top`/`Activity Monitor` to stay under your RAM ceiling.
